ClientOps Design System
Struktura plików i architektura CSS

Przenośny design system oparty na ClientOps Bootstrap, zmapowany z phoenix_resources/scss/theme/. Jeden import (index.css) daje tokeny Tailwinda, komponenty CSS i dark mode gotowy do użycia w każdym projekcie Laravel + React SPA.

Jak używać w nowym projekcie

/* resources/css/app.css */
@import "tailwindcss";
@import "./clientops-design-system/index.css";

Skopiuj cały folder clientops-design-system/ do nowego projektu — jeden import daje wszystko.

Dark mode — klasa .dark na <html>: 

document.documentElement.classList.toggle('dark')

Komponenty CSS → React (migracja do cva)

Klasy CSS z components/ są punktem startowym. Docelowo przenosisz je do komponentów React używając class-variance-authority. Po migracji komponentu — usuń odpowiadający plik CSS. 

// Zamiast: <button class="btn-mkds btn-primary btn-sm">
// Docelowo:
import { cva, type VariantProps } from 'class-variance-authority'

const button = cva('inline-flex items-center font-bold rounded', {
  variants: {
    variant: {
      primary:  'bg-primary-500 text-white hover:bg-primary-600',
      outline:  'border border-primary-500 text-primary-500 hover:bg-primary-500 hover:text-white',
      ghost:    'text-gray-700 hover:bg-gray-100',
      danger:   'bg-danger-500 text-white hover:bg-danger-600',
    },
    size: {
      sm: 'px-4 py-2 text-xs',
      md: 'px-6 py-[0.625rem] text-sm',
      lg: 'px-6 py-[0.875rem] text-base',
    }
  },
  defaultVariants: { variant: 'primary', size: 'md' }
})

// Użycie:
<button className={button({ variant: 'primary', size: 'sm' })}>Zapisz</button>
