Co już jest w ClientOps

Elementy zaprojektowane w schemacie bazy danych, stacku technicznym lub pipeline'ach kolejkowych.

SEO Audit Engine — baza danych

  • seo_analyses — status, score 0–100, gsc_connected, ga4_connected, pipeline statusy
  • seo_issues — severity (critical/warning/info), type, url, recommendation, assigned_to, fixed_at, previous_issue_id (regresja)
  • seo_cwv_snapshots (TimescaleDB) — LCP, CLS, INP, FCP, TTFB, performance_score, mobile/desktop, źródło pagespeed/gsc_field
  • seo_keyword_ranks (TimescaleDB) — pozycje fraz z GSC w czasie, clicks, impressions, CTR, per device
  • seo_monthly_snapshots — sesje organiczne, konwersje, CTR, avg position, top 3/10/50, backlinki nowe/utracone, referring domains
  • seo_backlinks — referring domain, target URL, anchor, domain_rating, is_toxic, first/last/lost_at
  • seo_competitors + seo_competitor_ranks (TimescaleDB) — tracking pozycji konkurentów per fraza
  • seo_keyword_opportunities — content gap, volume, difficulty, competitor_positions (JSON), has_dedicated_page, recommendation AI

Audyty zbiorcze

  • audits — type: seo | technical | ux | security | full, overall_score, pipeline statusy
  • audit_issues — category: ux | security | performance | technical | content | seo, severity, assigned_to, fixed_at
  • audit_scores — wynik per kategoria 0–100, details JSON (critical/warnings/info/passed)

Pipeline kolejkowy (zaplanowany)

  • CrawlWebsiteJob → FetchGSCDataJob + FetchGA4DataJob (równolegle) → AnalyzeKeywordsJob + AnalyzeSemanticsJob (AI) → GenerateAuditReportJob → NotifyAuditCompleteJob
  • Laravel Horizon — monitoring kolejek, balansowanie workerów
  • Osobne kolejki: crawlers, google-apis, ai-processing

Integracje Google (zaplanowane)

  • Google Search Console API — pozycje, kliknięcia, impressions, CTR
  • Google Analytics 4 API — ruch organiczny, konwersje
  • PageSpeed Insights API — Core Web Vitals (LCP, CLS, INP)

AI (zaplanowane)

  • Claude / OpenAI — analiza semantyczna treści, rekomendacje optymalizacji
  • AnalyzeKeywordsJob + AnalyzeSemanticsJob w pipeline
  • seo_keyword_opportunities.recommendation — generowane przez AI
  • audit_issues.recommendation — może być generowane przez AI

PDF Export

  • Laravel Browsershot — headless Chromium, async w kolejce
  • documents.pdf_path — ścieżka do PDF w S3/MinIO
  • Browsershot działa dla dokumentów TipTap — brak dedykowanych szablonów raportów audytowych

Reszta CRM (poza zakresem TODO v2.0.0 ale w ClientOps)

  • Leady + Kanban (7 kolumn) + lead_activities + konwersja do klienta
  • Klienci, Kontakty, Projekty, Oferty (dokument TipTap)
  • Tickety + SLA + sla_incidents (CheckSlaBreachesJob co 5 min)
  • Powiadomienia (email/SMS/in-app/WebSocket Reverb) z quiet hours
  • Multi-tenancy (subdomeny {slug}.clientops.pl, Global Scopes)
  • RBAC (Spatie Permission, 6 ról) + Policies per rekord
  • Audit log (Spatie Activity Log) — każda akcja z diffem

Czego brakuje — Faza 2 (krytyczne)

Moduły nieobecne w ClientOps, oznaczone jako KRYTYCZNE w roadmapie TODO v2.0.0. Bez nich system nie może wygenerować pełnych raportów audytowych.

PDF Raport Generator — 0% gotowości
ClientOps ma Browsershot (generowanie PDF z HTML), ale brak dedykowanych szablonów raportów audytowych. Potrzebne 3 typy: Prospekt SEO (8–15 stron, dla właściciela firmy, cel: zamknąć sprzedaż), Klient miesięczny (20–40 stron, pełne dane, ton partnerski), Ogólny audyt strony (10–18 stron, 4 kategorie). Każdy z tytułówką, tabelami, wykresami echarts, CTA.
Ahrefs / Semrush API — 0% gotowości
Tabele seo_backlinks, seo_competitors, seo_keyword_opportunities są w schemacie, ale nie mają źródła danych. Google Search Console nie daje Domain Rating, toxic score, content gap ani danych konkurentów. Bez Ahrefs/Semrush: sekcja backlinków w raporcie jest pusta, content gap niemożliwy, "wow efekt" prospektowy — brak.
UX / WCAG 2.1 Audit Engine — 0% gotowości
audits.type ma wartość ux i audit_issues.category ma ux, ale brak silnika sprawdzającego. Potrzebna integracja axe-core lub WAVE API. Zakres: kontrast kolorów, ARIA landmarks, nawigacja klawiaturą, tap targets (mobile), formularze (labele, błędy), popupy, font size, responsive layout.
Security Audit Engine — ~10% gotowości
audit_issues.category = security istnieje + seo_backlinks.is_toxic. Brakuje: security headers (CSP, HSTS, X-Frame-Options — securityheaders.com API), wykrywanie wersji CMS (WordPress/Drupal/Joomla outdated), exposed admin panel (/wp-admin), malware scan (Sucuri API), RODO compliance (cookie consent, dane kontaktowe), mixed content HTTPS.

Braki w sekcjach raportów audytowych

Szczegółowe zestawienie 3 typów raportów z TODO v2.0.0 — co jest w ClientOps, a czego nie ma.

Audyt 1 — Prospekt SEO (8–15 stron)

Dla właściciela firmy. Cel: zaimponować, wzbudzić ból, zamknąć sprzedaż.

  • Strona tytułowa — logo klienta + agencji, domena, data, osoba, klauzula poufności. Szablon PDF z placeholderami — brak.
  • Executive Summary — score per URL jest. Brak: agregacja /100, 3–5 krytycznych problemów, "dlaczego teraz".
  • Widoczność w Google — GSC (frazy top 3/10/50, trend). Brak: porównanie z 2–3 konkurentami (Ahrefs).
  • Techniczne SEO — 30 checków + PageSpeed. Gotowe.
  • On-page SEO (próbka) — title, meta, H1, treść, obrazy, linki. Gotowe.
  • Słowa kluczowe — content gap, branded, lokalne, long-tail. Brak (wymaga Ahrefs/Semrush).
  • Profil linków — seo_backlinks w schemacie. Tylko Open PageRank. Brak DR, toxic, trend 6m.
  • Quick wins tabela — seo_issues z impact_score. Brak sformatowanej tabeli w PDF.
  • CTA / Następne kroki — oferta, termin, kontakt, link do kalendarza. Brak.

Audyt 2 — Klient aktywny miesięczny/kwartalny (20–40 stron)

Pełny dostęp do danych. Ton: partnerski.

  • Dashboard KPI m/m + YoY — dane w seo_monthly_snapshots. Brak widoku porównawczego.
  • Widoczność i rankingi — queries w GSC. Brak: zmiany pozycji m/m, nowe/utracone top 10.
  • Ruch organiczny GA4 — GA4 jest. Brak: segmentacja, ścieżki konwersji, mobile vs desktop.
  • GSC pełna analiza — coverage, CWV, enhancements, linki wewnętrzne. Gotowe.
  • Techniczne SEO delta — historia jest (seo_monthly_snapshots). Brak delta view crawl m/m.
  • On-page postępy — checklist jest. Brak content gap.
  • Link building — nowe/utracone linki, DR trend, toksyczne, działania + plan. Brak (wymaga Ahrefs).
  • Monitoring konkurencji — tabela seo_competitors jest, brak alertów o zmianach pozycji/treści.
  • Zrealizowane prace (Work log) — logowanie prac: teksty, optymalizacje, linki, zmiany. Brak modułu.
  • Plan na następny miesiąc — 5 priorytetów z właścicielem + terminem. Brak modułu.
  • Komentarz stratega — pole tekstowe: interpretacja, ryzyka, szanse. Brak.

Audyt 3 — Ogólny audyt strony (10–18 stron)

4 kategorie. Ton: "widzimy ryzyko w 4 obszarach".

  • Executive Summary 4 kategorii — scoring SEO + Wydajność + UX + Bezpieczeństwo. Tylko SEO+Wydajność. Brak 2 kategorii.
  • Wydajność techniczna — CWV, PageSpeed API. Gotowe.
  • SEO stan podstawowy — 30 checków. Gotowe.
  • UX i dostępność — menu, breadcrumbs, tap targets, kontrast, ARIA, keyboard nav, formularze. Brak silnika (axe-core/WAVE).
  • Bezpieczeństwo — HTTPS jest. Brak: security headers, CMS wersje, malware scan, RODO, cookie consent.
  • Analityka — audyt narzędzi: GA4/GSC/GTM/konwersje/heatmapy na stronie klienta. Brak.
  • Treści i wizerunek — favicon jest. Brak: aktualność, blog, spójność, jakość zdjęć, błędy, 404.
  • Tabela priorytetów cross-category — SEO + Wydajność + UX + Bezpieczeństwo w jednej tabeli. Brak.
  • Metodologia + CTA — narzędzia, 3 opcje współpracy, termin, kontakt. Szablony brak.

Filar OwnCRM — całkowicie nieobecny w ClientOps

Poniższe moduły to unikalna wartość OwnCRM. W ClientOps nie istnieje ani jedna tabela, serwis ani komponent z tego filaru. Decyzja o integracji / migracji jest otwarta.

Moduł Co robi Status w ClientOps Priorytet wg TODO
Discovery Workshop
(9 kroków V2)		 AI-prowadzony workshop: wiedza o firmie → analiza konkurencji → persony → user journey → brand voice → strategia konwersji → IA → wireframe → copywriting. Każdy krok = dependency chain. Brak Krytyczny
AI Agents
(StepAgentService)		 Claude Sonnet jako persona 'Kasia'. Streaming SSE. MCP tools per step: save_knowledge(), update_step_output(), update_sitemap_pages(), update_block_content(). Output approval flow: proposed → approved/rejected. Brak Krytyczny
Knowledge System
(3 warstwy)		 Facts: pgvector 1536-dim, typed (decision/exclusion/requirement), confidence levels. ~242 per lead. Entities + Graph: relationship triples, wizualizacja grafu. Knowledge Items: tree via parent_id, flow: proposed → confirmed → export to Facts. Brak Krytyczny
Agent Knowledge Globalna baza materiałów referencyjnych z PDF-ów. Vector search. NIE per-lead (~1245 wierszy). Używana przez AI agenta jako baza wiedzy. Brak Krytyczny
Lead Classification
(8 wymiarów)		 Scorowanie per lead: business_type, audience_mix, content_scale, service_complexity, price_point, sales_cycle, trust_requirements, geographic_scope. Napędza automatyczny dobór block templates. Brak Krytyczny
Block Templates (~20) Szablony sekcji stron: hero (4 warianty), features (3), testimonials (2), pricing (3), CTA (3+). Każdy z fit_scores (0–1 per wymiar klasyfikacji), visual_weight, placement_rules. BlockComposerNode wybiera najlepsze per strona. Brak Krytyczny
Sitemap / IA Editor 3 widoki (drzewo, flat, wizualny), flat array z parentId, guessTemplate(). AI agent generuje IA, user edytuje w interfejsie drag & drop. Brak Wysoki
AI Visibility Score 8-czynnikowy scoring gotowości strony na AI search: schema.org, content depth, E-E-A-T signals, semantic HTML, FAQ, author markup, topical authority, structured answers. Unikalne — brak u konkurencji. Brak Krytyczny
SEO Workshop (4 kroki) Przegląd wyników → Lista zadań → Plan wdrożenia → Auto-wdrożenie przez WP REST API (Yoast meta tagi + alt tagi na obrazach). Unikalne — real-time zmiany na stronie klienta. Brak Krytyczny
Pipeline Builder (V1) Wizualny edytor (ReactFlow), 25+ node types (ai, conditional, loop, guardian, fact_saver, block_composer, gdrive_source, semantic_chunker...), BFS graph traversal. 18 gotowych pipeline'ów. Decyzja: migracja vs sunset. Brak Wysoki
Offer / Slide Deck Prezentacja oferty: canvas 1920×1080, CoverSlide, MethodologySlide, StepIntroSlide, PricingSlide, NextStepsSlide. PDF export przez jsPDF + html2canvas. ClientOps ma oferty jako dokument TipTap — inny format. Inne podejście Wysoki
MCP Tools 3 narzędzia dla AI agentów: get-seo-audit-overview, get-seo-page-detail, get-seo-task-list. AI może samodzielnie analizować audyty używając tych narzędzi. Brak Wysoki
Uwaga — pgvector
Knowledge System wymaga rozszerzenia PostgreSQL pgvector — nie ma go w stacku ClientOps. Dodaje typ vector(n) i operatory cosine similarity. Konieczne do Facts (1536-dim embeddingi OpenAI/Anthropic) i Agent Knowledge (vector search). Instalacja prosta — CREATE EXTENSION vector; — ale wymaga świadomej decyzji architektonicznej o dodaniu do stacku.
